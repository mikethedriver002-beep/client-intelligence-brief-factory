# Brief Factory Operating SOP

## Weekly rhythm

1. Monday: update source registry and manual inbox.
2. Tuesday: run Brief Factory in `draft_brief` or `full_delivery_packet` mode.
3. Tuesday/Wednesday: review QA report, source appendix, and top items.
4. Wednesday: polish the executive summary and recommended actions.
5. Wednesday/Thursday: send the agency/client packet.
6. Friday: tune sources and improve the template.

## QA rules

Every delivery should have:

- at least three useful items;
- source URLs on top items;
- source timestamps;
- confidence values;
- review flags for weak or uncertain items;
- a source appendix;
- a QA report;
- no auto-published content.

## Manual review posture

Automation reduces collection and formatting time. It does not replace editorial judgment. Treat every generated brief as a draft until reviewed.
